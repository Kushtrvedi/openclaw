# ANTI GENERIC ENGINE

## PURPOSE
Eliminate generic output completely

---

## RULE 1 — NO DEFAULT CONTENT

**FORBIDDEN PHRASES:**

- "we leverage AI"
- "seamless experience"
- "scalable solution"
- "market opportunity is huge"
- "innovative approach"
- "cutting-edge technology"
- "game-changing"
- "disrupt the industry"
- "best-in-class"
- "end-to-end"

**If detected → REWRITE**

---

## RULE 2 — FORCE SPECIFICITY

Every slide MUST include at least ONE of:

- **number** (e.g., "2-3% conversion rate")
- **example** (e.g., "Zerodha's journey...")
- **comparison** (e.g., "vs traditional brokers")
- **real scenario** (e.g., "User hesitates at $10k investment")

---

## RULE 3 — INSIGHT LAYER

Each section MUST answer:

→ **What is NOT obvious here?**

If answer is missing → REWRITE

**Example:**
- ❌ "AI helps users decide faster"
- ✅ "Users second-guess decisions within 30 seconds of hesitation—Fnomo intercepts that micro-moment"

---

## RULE 4 — CONTRAST THINKING

Each slide MUST show:

| Before | After |
|--------|-------|
| Static user journey | Adaptive decision layer |
| Post-decision doubt | Pre-decision validation |
| Generic recommendations | Personalized confidence |

---

## RULE 5 — DATA REQUIREMENT

At least 30% slides MUST include:

- tables
- charts
- structured data
- metrics with context

---

## RULE 6 — REJECTION CRITERIA

**If output looks like:**

- [ ] startup template (generic value props)
- [ ] LinkedIn post (buzzword-heavy)
- [ ] ChatGPT answer (surface-level)
- [ ] blog content (no structure)

→ **FAIL** → Re-write

---

## RULE 7 — McKINSEY STANDARD

Every slide must feel like:

- **INVESTOR DECK** (data + insight)
- **CONSULTING DECK** (structure + depth)
- **PRODUCT STRATEGY** (actionable + specific)

**NOT:**

- blog content
- generic pitch
- startup slide

---

## RULE 8 — ANTI-PATTERNS TO REMOVE

**Remove all:**

```
[ ] "Our mission is to..."
[ ] "We believe AI can..."
[ ] "The future of..."
[ ] "Leveraging technology..."
[ ] "Best-in-class solution"
[ ] "Seamless experience"
[ ] "End-to-end platform"
```

---

## RULE 9 — GENERATE ANGLES (BEFORE EXECUTION)

**Before outputting content, generate 3 unique angles:**

1. **Angle 1**: What's the counterintuitive insight?
2. **Angle 2**: What's the data reveal?
3. **Angle 3**: What's the hidden assumption?

**Pick best → Execute**

---

## RULE 10 — DEPTH TEST

**Before output, verify:**

- [ ] Does this include a number?
- [ ] Does this answer "so what?"
- [ ] Can someone act on this?
- [ ] Is this unique to this company?

If NO → Re-write
