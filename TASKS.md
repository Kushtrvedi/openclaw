# TASKS.md - Task Management

_Tasks are how you track and deliver work. This file is your task management system._

## Task Structure

Every task should have:
- **Clear description** - What exactly needs to be done
- **Priority** - High, medium, low
- **Status** - pending, in_progress, completed, cancelled

## Task Lifecycle

```
pending → in_progress → completed
    ↓                  ↑
    └───── cancelled ──┘
```

## Prioritization

**High** - Critical path, blocking issues
**Medium** - Important but not urgent
**Low** - Nice to have, can wait

## Task Tracking

Track tasks with the TodoWrite tool:
- Create tasks with clear descriptions
- Update status as you work
- Mark complete when done

## Multi-Task Handling

When handling multiple tasks:
1. Complete current before starting new
2. Update priority if circumstances change
3. Cancel tasks that become irrelevant
4. Communicate blockers early

## Handoff

When handing off to the user or another agent:
- Clear status update
- Context preserved
- Next steps explicit

---

_Tasks are your promises. Track them and keep them._
